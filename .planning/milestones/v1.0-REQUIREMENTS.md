## v1 Requirements

### Organization

- [x] **ORG-01**: User can create, edit, and delete workspaces
- [x] **ORG-02**: User can create, edit, and delete projects within a workspace
- [x] **ORG-03**: User can create, edit, and delete sections within a project
- [x] **ORG-04**: Sections have a user-defined order within a project
- [x] **ORG-05**: Workspaces, projects, and sections persist across page refresh

### Tasks

- [x] **TASK-01**: User can create a task with name (required), description, deadline (date/time), and tags
- [x] **TASK-02**: User can edit all fields of an existing task
- [x] **TASK-03**: User can delete a task (with confirmation)
- [x] **TASK-04**: User can toggle task status between OPEN and COMPLETED
- [x] **TASK-05**: User can assign/reassign a task to a section via menu
- [x] **TASK-06**: Tasks belong to a project and optionally to a section
- [x] **TASK-07**: Tasks have a user-defined order within their section/project

### Subtasks

- [x] **SUB-01**: User can create subtasks within a task (same fields as tasks)
- [x] **SUB-02**: Subtasks cannot have their own subtasks (one-level nesting enforced)
- [x] **SUB-03**: Attempt to create a sub-subtask is blocked with a clear message
- [x] **SUB-04**: Subtasks display within the parent task's expanded area

### Tags

- [x] **TAG-01**: User can create tags with a name and optional color
- [x] **TAG-02**: User can assign multiple tags to a task
- [x] **TAG-03**: User can remove tags from a task
- [x] **TAG-04**: User can view all tasks with a specific tag

### Views

- [x] **VIEW-01**: Project list view displays tasks grouped by section
- [x] **VIEW-02**: Project board view displays sections as columns with task cards
- [x] **VIEW-03**: User can toggle between list and board view per project
- [x] **VIEW-04**: Board view supports adding a new section as a column
- [x] **VIEW-05**: Task cards show: name, due date (if any), tags, completion toggle
- [x] **VIEW-06**: Task cards expand to show description and subtasks
- [x] **VIEW-07**: Task cards match dan-weinbeck.com card format (same styling tokens)

### Smart Views

- [x] **SMART-01**: Today view shows all tasks with deadline on today's date
- [x] **SMART-02**: Completed view shows all completed tasks (filterable by project)
- [x] **SMART-03**: Search finds tasks by name or description text
- [x] **SMART-04**: Filters & Tags view lists all tags and allows filtering by tag

### Navigation

- [x] **NAV-01**: Sidebar displays: Add Task, Search, Today, Completed, Filters & Tags
- [x] **NAV-02**: Sidebar displays workspace > project tree
- [x] **NAV-03**: Each project in sidebar shows open-task count
- [x] **NAV-04**: Add Task button opens a modal for quick task creation
- [x] **NAV-05**: Sidebar persists across page navigation (layout-based)

### Styling

- [x] **STYLE-01**: App replicates dan-weinbeck.com theme tokens (navy/gold/beige palette)
- [x] **STYLE-02**: App uses same fonts (Playfair Display, Inter, JetBrains Mono)
- [x] **STYLE-03**: Task cards use same card patterns as personal-brand site (rounded-2xl, shadow, hover lift)

### Data

- [x] **DATA-01**: All data persists in PostgreSQL via Prisma
- [x] **DATA-02**: Database schema supports the full hierarchy (Workspace > Project > Section > Task > Subtask)
- [x] **DATA-03**: Deleting a parent entity cascades to children (with confirmation dialog)

### Quality

- [x] **QUAL-01**: Lint passes (Biome)
- [x] **QUAL-02**: Build succeeds (Next.js)
- [x] **QUAL-03**: TypeScript compiles without errors
- [x] **QUAL-04**: Unit tests pass for subtask nesting constraint and Today filter logic
